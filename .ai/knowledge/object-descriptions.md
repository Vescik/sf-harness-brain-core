# Object Descriptions

What each object is and who owns it — the managed package or us. Facts only, never rules.
Entry format: `.ai/templates/knowledge-entry.md` (the optional "Keywords" field applies here —
terms from `keyword-taxonomy.md` only, never invented inline).

Also used by `suggest-test-cases` to expand technical vocabulary into business vocabulary
(HARNESS_BLUEPRINT.md section 11).

Scaling threshold (blueprint sections 3 and 8): at ~15-20 described objects, this file splits
into `.ai/knowledge/objects/<Name>.md` — one file per object, combining description + fields +
automation. Not built now.
